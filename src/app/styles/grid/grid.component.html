Grid system

The ob-grid class will create a dynamic 12 columns system. The number of columns shrinks to 8 below the LG breakpoint (1240px), to 4 below the SM breakpoint (600px) and to 2 below 360px. Items exceeding the column count will automatically be wrapped on a new line, so a single container suffices for all rows, albeit having one container per row is also possible. All items will always be aligned on grid lines and any unused space will be left blank.

There are two possibilities to span multiple columns, either by specifying the number of columns to span or by specifying the relative width. Absolute spans will span the exact number of columns given, trimmed to the maximum number of available ones, which can lead to empty space at a row's end. Relative spans will, when adding up to one, always occupy the whole row. For this to work, spans of one third had to be tweaked to be uneven when there are 8 or 4 columns. There's a caveat when 3 ob-third items have to span 4 columns, in this case, the ob-third-flexible class has to be manually set on the second or third ob-third item.

Flex system

The ob-flex class will create a dynamic 12 columns system. The number of columns shrinks to 8 below the LG breakpoint (1240px), to 4 below the SM breakpoint (600px) and to 2 below 360px. Each row will be filled with the maximum number of items. The items exceeding the column count will be automatically wrapped on a new line and the remaining space will be covered by evenly stretching all the items in the row. This behavior implies two things, first the items will not be aligned on the grid lines. Second, while one container for all items is possible, one container per row is advised to actually have some control over the number of items per row and their relative width.

Unlike with the ob-grid system, the items can't span multiple columns but they can be widened. The ob-grow-2 to ob-grow-6 classes are defined to extend an item's width relative to its siblings. The higher the growth factor, the wider the column. The actual width also depends on the number of other items in the row and their respective growth value. If there's enough space, ob-grow-6 will actually be 6 times wider than a default item. If not, then it will be as wide as possible.
