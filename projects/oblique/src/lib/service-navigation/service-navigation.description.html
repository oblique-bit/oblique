The ObServiceNavigationComponent is a regular Oblique component that aims at replacing the Header Widget. It proposes a collection of ready-to-use widgets, such as PAMS authentication, PAMS user profile, PAMS messages (notifications), PAMS applications, application info and language selection. Each widget can be individually shown or hidden and additional custom widget can be projected into it.

Integration

The ObServiceNavigationComponent should be located in the right corner of the header of your website. Oblique's master-layout provides the easiest way to integrate it. See more information in documentation of ObMasterLayout.

Structure

Controls of the ObServiceNavigationComponent are displayed from left to right as follows:

  1. Custom controls
  2. Message widget
  3. Info widget
  4. Applications widget
  5. Profile widget
  6. Authentication widget
  7. Language widget

For consistency reasons, the order is not adaptable.

Custom controls

The ObServiceNavigationComponent has the possibility to add additional controls with templates that have the #customControlTemplate-query.

For consistent styling, it's required to add the class .ob-header-control and Oblique's ObButton-directive to every projected button, otherwise the design will be off. 

<ng-template #customControlTemplate>
	... <!-- For example -->
	<button type="button" class="ob-header-control" mat-icon-button obButton="secondary">
		<mat-icon svgIcon="smile"></mat-icon>
	</button>
</ng-template>

Visibility of widgets

The language buttons are visible by default.

The widgets message, info, applications, profile and authentication are not visible by default.

Set the input of display[Widget] - property to true makes the widget or feature visible. For example:

<ob-service-navigation>
	....
	[displayMessage]="true"
	...
</ob-service-navigation>

Message widget

This widget is only displayed when the user is logged in with PAMS. It is a link to the ePortal notification page and shows how many unread messages are waiting in the user's inbox. The badge is only visible if the amount of unread messages is greater than 0.

Info widget

This widget opens a popover with project provided links and contact information. You should use the website's own help links and contact information for the best user support.

Applications widget

Is the user logged out, this widget is a link to the ePortal home page. Is the user logged in with PAMS, this widget opens a popover with a link to the ePortal home page as well as a list of favorite and last used ePortal applications that is delivered by ePortal's backend.

Maximum of list length

Change the maximum of the list-length with maxLastUsedApplications for the last used application list and maxFavoriteApplications for the maximum amount of favorite applications. So it shows the list length of last used and favorite applications, until the given maximum in the application widget popover.

Profile widget

This widget is only displayed when the user is logged in with PAMS. It shows the user's profile image if any and falls back to the user icon otherwise. It opens a popover with the logged in user information and an optional project provided link list.

You can add your custom profile-related links with the input profileLinks.

Authentication widget

The authentication widget shows a login button if the user isn't logged in. Is the user logged in, the login button is replaced with the logout button.

Language widget

This widget is for language selection. When more than 1 language is configured in the TranslateService, then 1 button per language will be displayed. For more configuration details see documentation of ObMasterLayout and Language.

Eportal CSRF interceptor

The ObEportalCsrfInterceptor will add a security token inside all your POST, PUT, PATCH and DELETE http calls to PAMS. This interceptor is mandatory for any application which communicates with PAMS, eportal's backend. It will be automatically added if you import either the ObServiceNavigationModule or the ObMasterLayoutModule in your project.
