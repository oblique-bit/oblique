This feature offers a nice way to display error messages related to a form element. The directive is meant for Material Design while the component is meant for Bootstrap.

Error messages

Oblique provides default error messages for common validation errors that can be overwritten in the project's translation file when necessary. All error messages defined in Oblique are described below. When available, the parameters that are passed on to the component are also given. These parameters can be used in the translation through interpolation.

The ErrorMessages directive should be put on a mat-form-field component and works in conjunction with a mat-error component, both from Angular. The mat-error component has to be present through empty and will be automatically populated with translations of thrown validation errors.

Note the overflow behaviour: long error messages will be shortened to available space, but will still be wholly displayed when hovered or when the related input has the focus.

Make sure the ErrorMessages directive is used, as the ErrorMessages component is meant for the Bootstrap theme.

If used within the orFormControlState directive, the component will automatically display the errors of the related control. In that case, the errors will be displayed in red, thanks to the has-error class set by the directive.

When used outside of the orFormControlState directive, the component can be placed anywhere on the page but then the source control has to be specified through the control input. In that case, no styling will be applied to the content.

Make sure the ErrorMessages component is used, as the ErrorMessages directive is meant for the Material theme.
